hexo + github 建立个人博客

2017-08-18
教程
hexo个人博客

早就有所耳闻使用GitHub可以搭建个人博客，但是网络上各种教程鱼龙混杂，几乎没有一个是完整的适合新人的教程。所以难免要遇到一些困难，这时切不可气馁，善于运用搜索引擎将会带来意想不到的效果。

好的，接下来是我建立blog的全过程，由于当时没有截图所以重新一步步又来一遍，并添加一些错误排查，内容不可能面面俱到，但希望本篇对读者有所帮助。

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
gitHub是一个面向开源及私有软件项目的托管平台，因为只支持git 作为唯一的版本库格式进行托管，故名gitHub。

hexo的安装与部署

安装npm

npm是nodejs官方为nodejs定制的一个工具，是Node.js的包管理器，是Node Packaged Modules的简称，通过npm可以下载安装nodejs的模块包，nodejs有很多优秀的模块包可以让开发这快速开发，省掉了很多步骤，可以直接使用npm安装使用。
详细安装过程请见这里，前提是git与node.js都已安装好。

检验是否安装成功,通过查看各种工具的版本信息检验各工具是否可以使用。使用 win+R 打开运行，输入cmd打开命令执行窗口，然后分别输入git --version node -v npm -v 回车查看个个工具的版本，若有报错，则重新安装或网上搜索答案。

安装hexo

首先选择一个喜欢的路径（比如我的是E:/hexo),打开文件夹在任意空白处右键，选择git bash here

然后在窗口中输入命令：npm install -g hexo (复制此代码后在窗口内使用ctrl+v失效，应使用右键后选择paste选项粘贴内容）其中 -g 指的是全局安装，等下会安装到指定目录下。

注意：此处似乎需要开vpn，如果出现被“墙”了情况请按如下步骤进行。
输入命令：npm install -g cnpm --registry=https://registry.npm.taobao.org 使用淘宝npm镜像安装

继续输入命令：cnpm install -g hexo-cli

然后：cnpm install hexo --save

检验是否安装成功：hexo -v

此部分详情请参照这篇文章的“安装hexo部分”

初始化部署hexo

部署进指定文件夹

在选择好的文件夹内任意空白处右键选择 Git Bash Here 输入命令 hexo init hexo将会部署到当前文件夹

注意：在此处等待时间较长，请耐心等待。出现warn请不用在意。

部署初始化后生成的一些文件：

生成器生成本地静态文件

输入命令：hexo g（hexo generate 的简写）本地生成静态网页，你会发现hexo根目录下多了public文件夹，此文件夹内就是将来上传至github的内容

启用本地查看服务

输入命令：hexo s（hexo server 的简写）开启本地预览服务，接下来在浏览器地址输入 localhost：4000就可以在本地查看当前部署好的blog

ctrl+c结束服务，结束后就不能打开localhost：4000

ok,hexo 本地部署结束，如果出现什么问题请利用好搜索引擎。

配置SSH key

使用用户名+密码使GitHub获取你的权限，然后可以允许你上传本地文件，这样并不安全。使用ssh key来解决本地和服务器的连接安全性更高。

基本设置

检验已存在的ssh

在hexo根目录下右键Git Bash Here，输入命令：cd ~/. ssh，若提示No such file or directory 说明你是第一次使用git，并没有这个文件夹。

手动的创建一个 .ssh文件夹即可，命令为：mkdir ~/.ssh

配置全局用户名和邮箱

配置全局的name和email，这里是的你github或者bitbucket的name和email

git config --global user.name "ledgil"

git config --global user.email "255986xxxx@qq.com"

获取ssh key

正常获取ssh key

输入命令 ssh-keygen -t rsa -C "邮件地址"（邮箱地址与上面的一样即可，这里只是添加一个凭证证明是你，而你的邮箱又是独一无二的，不用担心重复），连续按3次回车，就将密码设置为空，并创建了key。成功后会有这样的图案出现

然后找到用户目录找到.ssh文件夹，用编辑器打开id_rsa.pub文件，复制里面的内容。

创建key时可能会失败，如图

这是因为，电脑的.ssh文件夹可能没有写入权限，无法写入保存key,所以在一个有写入权限的地方生成key就行，详情请见这里。

在hexo根目录新建一个名字为getkey文件夹，输入以下命令

ssh-keygen -t rsa -C "邮件地址" -f getkey文件夹目录/key

打开getkey文件夹，用编辑器打开id_rsa.pub文件，复制里面的内容。

添加到github

点击右上角头像setting，左侧“ssh and gpg keys”,右上点“new ssh key”，title随便填，key粘贴复制的内容，左下点 Add SSH key。完成。

检测是否成功

输入 $ ssh -T 2559860328@qq.com (邮箱还是你的那个邮箱) 如果提示 Are you sure you want to continue connecting (yes/no)? 填入yes，
看到 Hi xxxxxx! You've successfully authenticated, but GitHub does not provide shell access. 说明SSH已配置成功！

独立域名绑定

我们可以用默认的 http://xxxxx.github.io访问我们的blog，想拥有一个属于自己的域名，当然也行。

注册域名

国内大大小小域名注册站点一大堆，推荐使用阿里云、腾讯等大公司，毕竟有保障。本人是在西部数码注册的.top域名，因为便宜啊，第一年只要4块钱。但是也要准备翻车的风险。

域名解析

绑定域名有带www和不带www的。域名配置最常见有CNAME和A记录，CNAME填写域名，A记录填写IP。因为不带www方式只能采用A记录，所以先ping一下你的用户名.github.io的IP（打开cmd 输入ping xxxxx.githb.io），然后到你的域名解析，将A记录指向你ping出来的IP，将CNAME指向xxxxx.github.io，这样可以保证无论是否添加www都可以访问

创建CANME文件

在hexo/source/里新建一个文本文件CNAME（无后缀名），编辑器打开填入你的域名，加不加www随意。无论当你访问带www还是不带www的域名，都会跳转到你在CNAME文件里填写的域名。

运行hexo g 和 hexo d 后相关配置就会部署并上传到GitHub。访问你的域名就可以看到blog。而默认的xxxxx.github.io并没有失效，访问该域名后会跳转到你的域名。
