hexo 个人博客使用技巧

2017-08-19 hexo 个人博客 阅读量

在前一篇文章中咱们利用GitHub和hexo快速建立了自己的个人博客，使用的是默认主题、发布了第一个hello world文章、连接上了我们的独立域名。这篇文章主要扩展一些hexo博客使用技巧。包括更换主题、主题设置、写作、配置修改等等。内容不可能面面俱到，但希望对读者有所帮助。

注意：在本地预览效果或者上传到GitHub之后，清理一下浏览器的缓存，否则可能看不到效果

主题

下载主题

我们可以使用官方主题,点击浏览后，找到GitHub图标进入开发者的GitHub，这里以主题hiker为例，获取下载链接

在hexo根目录下打开Git Bash Here，输入命令：git clone https://github.com/iTimeTraveler/hexo-theme-hiker.git themes/hiker （clone后的链接为你获取的下载链接，themes/hiker表示下载到themes/hiker文件夹）

下载时显示进度和下载速度，下载完成后将出现hexo目录下的themes文件夹内

配置主题

用编辑器打开hexo根目录下的_config.yml，找到theme，更改为主题名称即可

hexo g hex d 后访问域名查看效果

主题内容新建

新建页面

  • 新建关于页面

之前打开blog的一些页面比如about（关于我）页面会404 not find file。是因为本地没有配置并生成这些页面。

在根目录下Git Bash Here 打开命令窗口，输入命令： hexo new page "about" ，在根目录的source文件夹内就会显示一个about文件夹，打开编辑其中的 index.md 即可完成。

然后打开主题的配置文件，将About前的 # 去掉（没有 # 则不用动，没有About则给它追加上）,以下的 分类 标签 页面亦是如此，就不配图了。

hexo g 生成本地静态文件，hexo会根据配置将.md文件渲染成html文件，出现在public文件夹里。
hexo server 开启本地查看服务，访问localhost:4000可在本地查看部署好的blog

  • 新建分类页面

同理，在根目录下Git Bash Here 打开命令窗口，输入命令： hexo new page "categories" ，在根目录的source文件夹内就会显示一个categories文件夹，打开编辑其中的 index.md ，添加 type: "categories" 这使得hexo辩认出是分类页面，添加 comments: false 这使得该页面的评论功能关闭，改为true或者删掉这一行评论功能打开。保存。

类似于 关于 页面配置好主题的设置

然后 hexo g hexo server 在本地查看效果，没有反应则清一下缓存再查看，还没效果则在 index.md 添加 layout: "categories" 使hexo强制渲染本文件为分类。

  • 新建标签页面

同理，在根目录下Git Bash Here 打开命令窗口，输入命令： hexo new page "tags" ，在根目录的source文件夹内就会显示一个tags文件夹，打开编辑其中的 index.md ，添加 type: "tags" 这使得hexo辩认出是标签页面，添加 comments: false 这使得该页面的评论功能关闭，改为true或者删掉这一行评论功能打开。保存。

类似于 关于 页面配置好主题的设置

然后 hexo g hexo server 在本地查看效果，没有反应则清一下缓存再查看，还没效果则在 index.md 添加 layout: "tags" 使hexo强制渲染本文件为标签。

  • 新建自定义页面

Hexo 博客中所见文章都是经由渲染的静态网页，而静态网页的样式都直接由 Hexo 的主题控制，所以 Hexo 博客大部分都呈现出一种高度的统一化与规范化。不过 Hexo 提供了跳过渲染功能，使得我们可以直接在博客中放入自定义网页。

新建博客文章

  • 使用语法

上篇文章我们发布了第一篇blog，而使用的语法正是 Markdown ，关于Markdown的基本使用请参考本人的这篇文章，讲解了基本的语法，已经够用。

如果想了解更多语法，请参考这篇文章

  • 新建文章

我们使用命令：hexo new "Name" （Name 为文章名字），就会在hexo/source/_posts里出现 Name.md 的文件，使用编辑器和Markdown语法编辑即可。

或者直接在hexo/source/_posts文件夹里新建 Name.md 亦可。
