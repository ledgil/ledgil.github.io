发表于 2017-09-11 | 分类于 web安全

跨站脚本攻击(Cross Site Scripting)，为不和层叠样式表(Cascading Style Sheets, CSS)的缩写混淆，故将跨站脚本攻击缩写为XSS。恶意攻击者往Web页面里插入恶意Script代码，当用户浏览该页之时，嵌入其中Web里面的Script代码会被执行，从而达到恶意攻击用户的目的。xss的力量不可小觑！

本系列文章是本人阅读《xss跨站脚本攻击剖析与防御》并参考网易网课总结出的，对于一些难理解的东西仍然迷惑。本人拙笔，还请见谅，如有见解，欢迎评论。文章会持续优化更新，希望越来越完善。

xss简介

什么是xss

跨站脚本攻击(Cross Site Scripting)，为不和层叠样式表(Cascading Style Sheets, CSS)的缩写混淆，故将跨站脚本攻击缩写为XSS。恶意攻击者往Web页面里插入恶意Script代码，当用户浏览该页之时，嵌入其中Web里面的Script代码会被执行，从而达到恶意攻击用户的目的。常常由于web应用程序对用户的输入过滤不足而产生。

xss危害

  • xss可以盗取用户cookie信息，从而获取用户隐私信息，并利用用户身份进一步操作。
  • 会话劫持，导致强制发表日志、发送邮件等。
  • 钓鱼欺骗，盗取各类账号。
  • 制造蠕虫
  • 弹出广告、页面，恶意刷流量
  • 网页挂马
  • 结合其他漏洞，如csrf漏洞
  • 控制受害者机器向其他网站发起攻击
  • 等等

之前各种部门对xss并不重视，认为xss就是弹个窗而已，没有什么危害，但是xss被一些精明的黑客利用，其危害已经与sql注入旗鼓相当，甚至更厉害。

    基本了解火狐浏览器的界面、设置、插件查找安装。

    JavaScript 是 Web 的编程语言，所有现代的 HTML 页面都使用 JavaScript。

xss分类

反射型xss

反射型xss简介

反射型xss也叫非持久性xss，主要用于将恶意脚本附加到url地址的参数中，例如：http://www.test.com/search.php?key="><script>alert("xss")</script>

反射型xss一般是攻击者使用特定的手法（比如电子邮件），诱导用户点击一个恶意的链接，用户点击之后，恶意的JavaScript代码就会在浏览者的主机上运行。这类xss通常出现在网站的搜索栏、用户登入口等地方。

反射型xss测试

在xss（reflected）模块的输入框中输入任意内容提交，其将显示我们刚刚输入的内容。查看源代码发现，它将我们输入的内容直接输出。那么我们就可以利用。

输入<script>alert("xss")</script>发现弹窗，这说明恶意代码植入成功。

后台php文件将<script>alert("xss")</script>直接输出，经过渲染成html网页，而这个html标签执行了alert命令。

持久型xss

持久型xss简介

持久型xss也叫做存储型xss，这类xss不需要用户点击链接就能够执行脚本。攻击者事先将恶意的JavaScript代码上传或存储到服务器中，只要受害者浏览包含此恶意代码的页面就会执行恶意代码。

此xss多出现在网站的评论区、留言板、博客日志等与数据库有交互的地方。当恶意脚本存储到数据库中，有其他用户或者管理员访问这个网站，站点就会读取恶意脚本执行。

持久型xss测试

这是一个类似留言或者评论的板块，我们试着正常留言任意内容，提交后将会存储到数据库中。

当我们以后再访问这个网页时，就会从服务器读取我们的这条内容，然后展示到页面上，即会一直存在。

我们输入测试代码<script>alert("xss")</script>,这里可以在Name和Message框中都可以输入，因为并没有字数和类型限制，这里仅以Message框作为测试。

当刷新页面或者重新进入这个网页，就会弹窗。说明已经存储到了数据库中。
我们一进入这个页面就会执行xss，我们点击左侧的Reset DB，点击下方的按钮就可以重置数据库，这时我们保存的内容就被删除。

DOM型xss

DOM就是指文档对象模型，使得程序和脚本可以动态访问和更新文档内容、结构和样式。
客户端的脚本程序可以通过DOM动态检查和修改页面的内容，它不依赖服务器的数据，而是从客户端获得DOM中的数据并在本地执行。

xss cheat sheet

在测试xss的过程中，我们常常需要尝试各种xss输入，以绕过服务端程序对跨站脚本的过滤。所以就会有慈悲之人整理好各种可能用到的xss语句。

xss测试工具

火狐浏览器插件

firebug

firebug提供了许多功能，包括动态修改HTML代码、探索DOM结构、监视网络请求和响应、调试和监测任何页面的css、html和JavaScript等。

打开火狐浏览器，点击右上角的菜单，点击附加组件，右上角搜索框中搜索firebug安装。按下f12键就可以调出firebug。
或者点击此处，直接下载安装即可。

tamper data

tamper data能够抓取http请求，它可以查看和修改http/https的头部和post数据。

点击此处下载并安装，在火狐浏览器的工具菜单（右键点击浏览器上方空白处）里选择 “tamper data”，就会弹出窗口。

proxy switcher

proxy switcher是一个快速配置代理的插件，点击此处下载安装，在工具栏上打开，界面如图，在menu内就可以配置代理，方便与接下来的代理抓包软件配合抓包。

当然可以切换至“no proxy”或“system proxy”来关闭代理或使用系统的代理（vpn）。

软件

Fiddler

Fiddler是一款代理抓包工具，是功能很强大的HTTP/https网络调试器，该工具能够记录和监测所有客户端和服务器之间的http通信，此外还提供了清除IE缓存、请求构造器、文本转换等功能。

点击此处进入Fiddler官网选择免费下载，安装成功后界面如图所示。
